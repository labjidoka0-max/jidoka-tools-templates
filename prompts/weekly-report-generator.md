# 週次レポート 自動文章化プロンプト

## 用途
Excelの数値表から、そのまま上司に出せる週次レポートの文章を自動生成。
60分→2分に短縮。

## Claude Codeへの指示（コピペ用）

```
以下の週次売上データを元に、上司提出用の週次レポート文章を書いてください。

【出力構成】
1. 今週のサマリー（1段落、3文以内）
2. 前週比・前年同週比の分析（定量）
3. 要因分析3点（上位要因の推測）
4. 来週のアクション（2〜3点）

【トーン】
- 敬体（です/ます調）
- 過度に楽観も悲観もしない
- 数字は具体的に

【データ】
| 週 | 売上 | 客数 | 客単価 |
|----|------|------|--------|
| 先々週 | 1,200万 | 800人 | 1.5万 |
| 先週 | 1,450万 | 890人 | 1.63万 |
| 今週 | 1,380万 | 920人 | 1.5万 |
```

## 返ってくる出力例

```
## 今週の週次レポート

今週の売上は1,380万円となり、先週比で70万円減（▲4.8%）となりました。
一方で客数は920人と先週比30人増（+3.4%）しています。
客単価が1.5万円へ下落（先週1.63万円）したことが売上減の主因と考えられます。

【要因分析】
1. 高単価商品の販売数減少（データ別途要確認）
2. 新規顧客比率の上昇による単価低下
3. ...

【来週のアクション】
- 客単価回復のため、アップセル施策の再徹底
- 新規顧客へのセット販売提案の強化
```

## 応用

- 英文対応: 「英語版も作って」
- 図表付き: 「Markdownでグラフ用データも添える」
- Slack用: 「3行以内のSlack共有用も」
- 他部署展開: 「経営会議用の3分プレゼン台本も」

## コツ

データが3期間以上あると、Claude Codeが時系列トレンドを拾いやすい。
「過去4週のデータ」があれば精度が上がります。
